This project is a cool one. In thinking that I wanted to make a game out of something from the start, I was SO excited when I found the ColourLovers API. "A database of random names that go with colors? I think I could do something like Wordle" was my first thought, and honestly I'm proud to say it didn't end up being a clone of it! There are the basic components like limited guesses and whatnot that align of course, but I feel confident in it actually standing on its own as a pretty fun game, especially since I don't see a lot of good browser games being made anymore. The Requirements

Needless to say I chose my own API to run with for this project. With the full functionality of my game, which I'll go more in-depth on, I fully believe my project is at least roughly equivalent to what was asked of other students. As I spoke about with Professor Wheeland, my API is NOT CORS-enabled, however I found a method through jQuery to grab information from the API nonetheless and therefore the API was allowed. While my project may not have a search functionality like the more-trodden paths that this project provided, local storage functionality between user visits is shown through the user's Game History on the bottom of the main page of the site. The site also has navigation controls in the game's color picker, and straight-forward buttons for menu traversal. I actually attempted to have another user control in-game where the color would change if you were to change the values in the input boxes next to the sliders, but due to the Color Picker not being my code I couldn't yet find a way to do that. When it comes to design & interaction, I feel my project is incredibly straight-forward and easy for a user to understand, and CERTAINLY doesn't resemble the infamous Giphy Search UI. There are no places where errors could occur unless users have a very poor internet connection after the main site itself loads (and before the API info. is grabbed). It stays incredibly responsive to different aspect ratios, and uses flexbox for multiple of its menus. I have also overlooked everything regarding code conventions, and should have no issues.

The Grade
